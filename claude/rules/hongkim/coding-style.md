# Coding Style

## 불변성 (CRITICAL)

객체를 직접 수정하지 말고 항상 새 객체를 반환.

## 파일 구성

- 200~400줄 권장, 최대 800줄
- 기능/도메인별로 구성

## 임시 작업 공간: .claude/tmp/

세션 중 `.claude/tmp/`를 적극적으로 활용:
- 코드 스니펫 초안, 일회성 스크립트, 중간 결과물, 작업 노트 등
- 전역 git-ignored이므로 프로젝트 오염 없이 자유롭게 사용 가능

## 작업 완료 체크리스트

- [ ] 함수 50줄 이하
- [ ] 파일 800줄 이하
- [ ] 중첩 4레벨 이하
- [ ] 하드코딩 없음
- [ ] 불변 패턴 사용
