# Coding Style

## 객체지향 원칙

- **SRP** (단일 책임): 클래스·함수는 하나의 책임만 가짐
- **OCP** (개방-폐쇄): 확장에는 열림, 수정에는 닫힘 — 기존 코드 변경 없이 기능 추가
- **LSP** (리스코프 치환): 하위 타입은 상위 타입을 완전히 대체 가능해야 함
- **ISP** (인터페이스 분리): 사용하지 않는 메서드를 강제하는 인터페이스 금지
- **DIP** (의존성 역전): 구체 구현이 아닌 추상(인터페이스)에 의존
- **DI** (의존성 주입): 객체를 직접 생성하지 말고 외부에서 주입받음

---

## 불변성

객체를 직접 수정하지 말고 항상 새 객체를 반환.

---

## 단순성 원칙 (Simplicity First)

- 요청에 명시된 것만 구현 — 추측성 기능 추가 금지
- 조기 추상화 금지 — 3개 이상 반복될 때 추상화 고려
- 발생하지 않는 시나리오에 대한 에러 처리 금지
- 미래 요구사항을 위한 확장 포인트 선제 추가 금지

---

## 파일 구성

- 200~400줄 권장, 최대 800줄
- 기능/도메인별로 구성

---

## 작업 완료 체크리스트

- [ ] 함수 50줄 이하
- [ ] 파일 800줄 이하
- [ ] 중첩 4레벨 이하
- [ ] 하드코딩 없음
- [ ] 불변 패턴 사용

체크리스트 항목 미충족 시 → 해당 항목 수정 후 완료 보고. 미충족 상태로 완료 보고 금지.
