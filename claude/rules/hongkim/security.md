# Security Guidelines

## 커밋 전 보안 검증

### 자동 스캔 (Bash 명령)

검증 가능한 모든 항목은 명령으로 실행. 결과 0건 확인 후 통과.

| 검증 항목 | 명령 |
|---|---|
| 시크릿 패턴 | `rg -i "(api[_-]?key\|secret\|token\|password)\s*=\s*['\"]"` |
| SQL 문자열 concat | `rg "(execute\|query)\s*\(.*[\"'].*\+"` |
| 하드코딩 IP/URL | `rg "https?://[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+"` |
| 환경변수 로드 누락 | `rg "process\.env\." -l \| xargs grep -L "dotenv\|getenv"` |

### 수동 검토 체크리스트

**공통 (모든 프로젝트)**:
- [ ] 에러 메시지에 민감 정보 노출 없음

**백엔드/API 변경 시 추가**:
- [ ] 모든 사용자 입력 검증됨
- [ ] 인증/인가 검증됨 (자동 스캔으로 잡히지 않는 비즈니스 로직)

**웹 앱 (브라우저 노출) 변경 시 추가**:
- [ ] XSS 방지 (HTML sanitize 적용)
- [ ] CSRF protection 활성화됨
- [ ] 모든 엔드포인트에 rate limiting 적용됨

## 시크릿 관리

- 소스 코드에 시크릿 하드코딩 금지
- 환경 변수 또는 secret manager 사용
- 앱 시작 시 필수 시크릿 존재 여부 검증
- 노출 가능성 있는 시크릿은 즉시 로테이션

## 보안 이슈 심각도 분류

| 심각도 | 기준 | 대응 |
|---|---|---|
| CRITICAL | 시크릿 노출, 인증 우회, RCE 가능 | 즉시 작업 중단 → 프로토콜 실행 |
| HIGH | SQL injection, XSS, CSRF 취약점 | 즉시 작업 중단 → 수정 후 재개 |
| LOW | 에러 메시지 노출, 불필요한 권한 | 다음 커밋 전까지 수정 |

## 보안 이슈 대응 프로토콜

**Trigger**: 위 체크리스트에서 미통과 항목 발견 OR 코드 분석 중 취약점 탐지.

**Workflow**:
1. 즉시 작업 중단
2. 심각도 분류 (위 표 기준)
3. CRITICAL/HIGH → `security-review` 스킬 사용
4. 노출된 시크릿 로테이션 안내:
   - LLM 직접 수행 불가 → 사용자에게 즉시 알림
   - 알림 내용: 노출 위치, 영향 범위, 로테이션 우선순위
   - git history 내 노출 여부 확인: `git log -p -S {시크릿 일부}`
5. 전체 코드베이스에서 유사 패턴 검색:
   - 위 자동 스캔 명령을 전체 코드베이스 대상으로 재실행
   - 발견 시 동일 프로토콜 적용
6. 수정 후 자동 스캔 + 수동 체크리스트 재검증

**금지**:
- 시크릿 노출 발견 후 사용자 알림 생략
- "별일 아닌 것 같음" 판단으로 LOW 처리
