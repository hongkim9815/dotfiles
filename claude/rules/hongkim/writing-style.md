# Writing Style Rule
<!-- 문서 작성 시 독자(AI vs Human)에 따라 스타일 선택 -->

## 독자 분류

| AI 독자 | Human 독자 |
|---|---|
| CLAUDE.md, SKILL.md, rules, plans, agents | README.md, wiki, design doc, changelog |

**README.md는 항상 Human 독자용. 예외 없음.**

기타 문서 판단 기준:

| 시그널 | 독자 판단 |
|---|---|
| 파일명이 `README`, `CHANGELOG`, `wiki`, `design-doc` 포함 | Human |
| 파일명이 `CLAUDE`, `SKILL`, `rules/`, `plans/`, `agents/` 포함 | AI |
| 사람이 직접 읽는 것이 주목적인 문서 (온보딩, 가이드, 릴리즈 노트) | Human |
| Claude가 실행 지침으로 로드하는 문서 | AI |
| 위 기준으로 판단 불가 | 사용자에게 확인 |

## AI 독자용 문서 원칙

- **명사형 종결 어미 우선**: `~임`, `~함`, 동사 명사형 (`진행`, `확인`), `~금지`
- `~할 것`은 맥락상 명확히 필요한 경우에만 허용
  - 허용: MANDATORY 강조, 예외 없는 절대 조건 (`반드시 ~할 것`)
  - 금지: 일반 서술 (`X를 사용할 것` → `X 사용`)
- **명사형 표현**: `X를 사용함`, `X 사용 금지` (서술형 금지: `X를 사용합니다` X)
- **동일 개념 = 동일 단어**: 용어 변형 금지 (`인증` / `auth` / `로그인` 혼용 X)
- **대명사 최소화**: "it", "이것", "해당" → 명사 반복이 정확함
- **제약 경계 명시**: 해야 할 것 + 하면 안 될 것 모두 명시
- **조건 선행**: `X 상태일 때 → Y 실행` (조건 → 행동 순서)
- **섹션 자기완결성**: 각 섹션을 단독 발췌해도 의미 통하게 작성
- **산문 최소화**: 불릿 리스트·표 우선, 중요 정보는 상단 배치

## Human 독자용 문서 원칙

- **mermaid 다이어그램**: 관계·흐름이 3개 이상이면 mermaid 사용
  - 아키텍처 → `flowchart` / `graph`
  - API 흐름 → `sequenceDiagram`
  - 상태 전이 → `stateDiagram-v2`
- **표(Table)**: 비교·옵션·설정·파라미터에 사용
- **TL;DR 블록**: 긴 문서는 상단에 한 줄 요약 필수
- **`<details>` 접기**: 심화 내용·트러블슈팅·고급 옵션 계층화
- **현실적 예시 데이터**: `alice@example.com`, `user_42` (X: `foo`, `test@test.com`)
- **Why 섹션**: 설계 결정에 이유 설명

---

## 새 문서 작성 시

1. 독자 분류 판단 (위 기준표 적용)
2. 해당 독자용 원칙 섹션 참조하여 작성
3. 판단 불가 시 → 사용자에게 확인 후 진행

## 기존 문서 수정 시

덮어쓰기 전 반드시 사용자에게 확인:
- AI용 / Human용 중 어느 방향으로 최적화할지
- 덮어쓸지, 별도 파일로 저장할지
