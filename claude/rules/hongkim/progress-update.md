# Progress Update

진행 상황 보고에 대한 규칙. 말투·어휘는 `speech-style.md` 참조.

## 즉각 업데이트 (short-term)

현재 진행 중인 작업의 즉각적 상태.

**발동 트리거**:

| 트리거 | 행동 |
|---|---|
| 첫 번째 tool call 직전 | 하려는 작업을 한 줄로 설명 |
| 버그·이슈 발견 | 발견 내용 1~2줄 보고 |
| 방향 전환 (초기 접근 포기) | 변경 이유 + 새 접근 1줄 |
| tool call 3회 이상 연속 예상 | 작업 그룹의 목적 1줄 |
| 단일 tool call이 장기 실행 예상 | 예상 소요·진행 방식 1줄 |
| 예상과 다른 결과 수신 | 결과·다음 행동 1줄 |

## 누적 맥락 업데이트 (long-term)

**발동 트리거**:

| 트리거 | 행동 |
|---|---|
| 작업 단계 완료 (subtask 종료·파일 그룹 처리 완료 등) | 단계 결과 요약 + 다음 단계 명시 |
| 사용자 입력 직후 첫 응답 시작 전 (이전 세션과 연결 필요한 경우) | 이전 맥락 1~2줄 재요약 후 본 응답 시작 |

**내용 요건**:
- 맥락 재파악 가능하도록 충분한 정보 포함 (다음 세션에서도 단독으로 의미 통하게)
- 내부 코드명·약어 사용 금지 — 완전한 명사로 표현
- 사용자가 마지막 메시지 이후의 맥락만으로 다음 행동 결정 가능해야 함

## 금지

- 사고 과정 narration ("음 이걸 보고 있는데…", "잠시 생각해보면…") 금지
- "검토하겠음", "확인하겠음" 같은 의도만 표명하고 결과 없는 업데이트 금지
- 동일 정보 반복 업데이트 금지
